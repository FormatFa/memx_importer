# 微信支付账单转Memx导入工具

这是一个用于将微信支付导出的Excel账单文件转换为Memx记账软件(https://moneymanagerex.org/)可导入的CSV格式的Go程序。

## 功能特性

- 解析微信支付导出的xlsx格式账单文件
- 根据配置文件映射收款人到相应的类目和子类目
- 根据配置文件映射支付方式到特定的账号名称
- 支持严格模式：未找到映射时停止并提示用户补充配置
- 输出符合Memx记账软件格式的CSV文件

## 安装依赖

```bash
go mod tidy
```

## 使用方法

```bash
go run main.go <微信Excel文件> <输出CSV文件> [配置文件]
```

参数说明：
- `微信Excel文件`: 微信支付导出的xlsx格式账单文件
- `输出CSV文件`: 转换后的CSV文件路径
- `配置文件`: 可选，默认为 `config.yaml`

## 示例

```bash
# 使用默认配置文件
go run main.go "微信支付账单流水文件(20251029-20251129)_20251129211404.xlsx" output.csv

# 使用自定义配置文件
go run main.go "微信支付账单流水文件(20251029-20251129)_20251129211404.xlsx" output.csv my_config.yaml
```

## 配置文件格式

配置文件使用YAML格式，包含以下内容：

```yaml
# 是否启用严格模式
# true: 如果找不到映射，程序会报错停止
# false: 如果找不到映射，相应字段会留空继续处理
strict_mapping: true

# 收款人到类目、子类目的映射表
category_mappings:
  # Shopping
  - payee: "京东"
    category: "购物"
    subcategory: "网购"

# 支付方式到账号的映射表
account_mappings:
  - payment_method: "零钱"
    account: "微信零钱"
  - payment_method: "/"
    account: "现金"
  # 添加更多映射...
```

## 数据映射关系

微信支付Excel字段 → CSV输出字段：

- 交易时间 → 日期 (YYYY-MM-DD格式)
- 交易单号 → ID
- 收/支 → 类型
- 交易对方 → 收款人
- 金额(元) → 金额
- 当前状态 → 状态 (固定为"已核实")
- 商品名称 → 备注
- 商户单号 → 账号 (原始数据，供参考)
- 支付方式 → 账户 (原始支付方式)
- 支付方式 → 账号 (映射后的简化账号名称)

### 映射关系
- **类目映射**: 根据收款人名称映射到相应的类目和子类目
- **账号映射**: 根据支付方式映射到简化的账号名称

## 注意事项

1. 确保微信支付Excel文件格式正确，数据从第17行开始（第16行为表头）
2. 如果启用严格模式，需要确保所有收款人都在配置文件中有对应映射
3. 程序会自动处理金额字段中的¥符号
4. 日期格式会自动转换为YYYY-MM-DD格式

## 错误处理

- 如果找不到收款人的类目映射且启用了严格模式，程序会停止并显示缺失的收款人名称
- 此时需要在配置文件中补充相应的映射关系后重新运行
